# 文档站说明

这里是楸晟的个人文档站，用来保存可复用、可检索、可长期维护的内容。

## 内容范围

- 技术学习笔记和课程
- 项目背景、方案和复盘
- 部署、运维和工具链记录

## 写作约定

每篇文档尽量说清三个问题：

1. 这件事解决什么问题
2. 当前采用什么做法
3. 后续维护时需要注意什么

这样以后回来看，不只是看到命令和配置，也能理解当时为什么这样做。

## 新增项目

每个项目单独建一篇文档，按这个模板写：

```md
# 项目名称

## 背景
这个项目要解决什么问题？

## 当前方案
现在采用什么技术栈、架构或流程？

## 部署方式
如何构建、发布和回滚？

## 维护记录
重要变更、故障处理和后续计划。
```
